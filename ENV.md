# ENV.md — Обязательные переменные окружения

Все переменные задаются в Render → Environment (или в `.env` при локальном запуске).

## Обязательные

| Переменная | Пример | Описание |
|---|---|---|
| `BOT_TOKEN` | `1234567890:AAF...` | Токен Telegram-бота (из @BotFather) |
| `BOT_USERNAME` | `ege_helper_bot` | Username бота **без @** (нужен для реф-ссылок) |
| `ADMIN_IDS` | `123456789,987654321` | ID администраторов через запятую |

## YooMoney

| Переменная | Пример | Описание |
|---|---|---|
| `YOOMONEY_RECEIVER` | `4100116075156746` | Номер кошелька/получателя YooMoney |
| `YOOMONEY_WEBHOOK_SECRET` | `some-long-secret-string` | Секрет для проверки входящих webhook-запросов |
| `PUBLIC_BASE_URL` | `https://ege-bot-2etu.onrender.com` | Публичный URL сервиса (используется в `successURL`) |

## Дополнительные (если используются)

| Переменная | Описание |
|---|---|
| `POLZA_AI_API_KEY` | API-ключ для AI-разбора фото задач |

## Переменные, которые можно удалить

Следующие переменные больше не используются (LAVA удалён):
- `LAVA_SHOP_ID`
- `LAVA_SECRET_KEY`
- `LAVA_API_KEY`

## Настройка YooMoney webhook на Render

После деплоя настрой уведомления в личном кабинете YooMoney:

- **URL**: `https://<твой-домен>/yoomoney-webhook`
- **Метод**: POST
- **Заголовок**: `X-Yoomoney-Secret: <YOOMONEY_WEBHOOK_SECRET>`

> Если YooMoney не поддерживает кастомные заголовки, оставь `YOOMONEY_WEBHOOK_SECRET` пустым —
> тогда webhook-эндпоинт будет принимать все запросы без проверки секрета.
