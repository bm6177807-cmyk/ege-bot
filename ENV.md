# ENV.md — Обязательные переменные окружения

Все переменные задаются в Render → Environment (или в `.env` при локальном запуске).

## Обязательные

| Переменная | Пример | Описание |
|---|---|---|
| `BOT_TOKEN` | `1234567890:AAF...` | Токен Telegram-бота (из @BotFather) |
| `BOT_USERNAME` | `ege_helper_bot` | Username бота **без @** (нужен для реферальных ссылок) |
| `ADMIN_IDS` | `123456789,987654321` | ID администраторов через запятую |

## YooMoney

| Переменная | Пример | Описание |
|---|---|---|
| `YOOMONEY_RECEIVER` | `4100116075156746` | Номер кошелька/получателя YooMoney |
| `YOOMONEY_WEBHOOK_SECRET` | `some-long-secret-string` | Секрет для проверки входящих webhook-запросов (sha1_hash) |
| `PUBLIC_BASE_URL` | `https://ege-bot-2etu.onrender.com` | Публичный URL сервиса (используется в `successURL`) |

## Render

| Переменная | Описание |
|---|---|
| `PORT` | Порт, на котором запускается aiohttp web-сервер. На Render проставляется автоматически. По умолчанию `10000`. |

## Дополнительные (если используются)

|---|---|
| `POLZA_AI_API_KEY` | API-ключ для AI-разбора фото задач |

## Настройка YooMoney webhook на Render

После деплоя настрой уведомления в личном кабинете YooMoney:

1. Войди в [кабинет YooMoney](https://yoomoney.ru/transfer/myservices/http-notification)
2. Укажи **URL уведомлений**: `https://<твой-домен>/yoomoney-webhook`
3. Укажи **секретное слово** (notification password) — то же, что в `YOOMONEY_WEBHOOK_SECRET`
4. Метод: **POST**

### Как работает проверка подписи (SHA-1)

YooMoney подписывает каждое уведомление хешем:

```
sha1(notification_type&operation_id&amount&currency&datetime&sender&codepro&**secret**&label)
```

Бот автоматически проверяет подпись по этой формуле.

Если `YOOMONEY_WEBHOOK_SECRET` не задан — подпись не проверяется (**не рекомендуется для продакшена**).

## Типовые проблемы

### TelegramConflictError: terminated by other getUpdates request

**Причина:** Запущено несколько копий бота с одним и тем же токеном (например, старый процесс на Render не завершился, или бот запущен локально одновременно с Render).

**Решение:**
1. Убедись, что нигде локально не запущен `python bot.py`
2. В Render → твой сервис → **Suspend** на 1 минуту, затем **Resume**
3. Если не помогло — сделай **ротацию токена**:
   - BotFather → `/revoke` (или "Revoke token") → получи новый `BOT_TOKEN`
   - В Render обнови переменную `BOT_TOKEN`
   - Redeploy

Ротация токена гарантированно "убивает" все старые инстансы, так как они продолжают стучаться старым (уже недействительным) токеном.

### YooMoney ссылка не открывается (404)

Убедись, что в коде используется `https://yoomoney.ru/quickpay/confirm?...`, а не `https://yoomoney.ru/quickpay/shop.xml?...`.

### Webhook возвращает 403

Проверь, что `YOOMONEY_WEBHOOK_SECRET` в Render совпадает с "секретным словом" из настроек YooMoney.

### Премиум не активируется после оплаты

1. Проверь в Render Logs наличие запроса `POST /yoomoney-webhook` и статус ответа
2. Если запроса нет — YooMoney не отправляет уведомления. Проверь настройки HTTP-уведомлений в личном кабинете
3. Если статус 403 — несовпадение секретного слова
4. Если статус 200, но премиум не выдаётся — проверь `label` в платежном URL (должен совпадать с `order_id` в `pending_payments`)
