# Содержимое кодификаторов ЕГЭ

В этой директории хранятся JSON-файлы с полной структурой кодификаторов ЕГЭ по предметам.

## Формат файла (v1)

Каждый файл соответствует одному предмету и называется по схеме `<subject>.v1.json`.

### Схема

```json
{
  "version": 1,
  "subject": "<subject_code>",
  "subject_name": "<Название предмета>",
  "themes": [
    {
      "id": "<theme_id>",
      "name": "<Название темы>",
      "parent_id": "<parent_theme_id или null>"
    }
  ],
  "exam_tasks": [
    {
      "id": "<exam_task_id>",
      "number": "<номер задания>",
      "name": "<Описание задания>",
      "part": "<часть: A/B/C или null>"
    }
  ],
  "mapping": [
    {
      "exam_task_id": "<exam_task_id>",
      "theme_ids": ["<theme_id>", ...]
    }
  ]
}
```

### Описание полей

| Поле | Обязательно | Описание |
|------|-------------|----------|
| `version` | ✅ | Версия формата. Сейчас только `1`. |
| `subject` | ✅ | Код предмета (`chemistry`, `biology`, `math`, …). |
| `subject_name` | — | Читаемое название предмета. |
| `themes` | ✅ | Список тем кодификатора. |
| `themes[].id` | ✅ | Уникальный идентификатор темы (например `1.1`, `2.3`). |
| `themes[].name` | ✅ | Название темы. |
| `themes[].parent_id` | — | `id` родительской темы для иерархии, иначе `null`. |
| `exam_tasks` | ✅ | Список типов заданий ЕГЭ. |
| `exam_tasks[].id` | ✅ | Уникальный идентификатор типа задания (например `task_1`). |
| `exam_tasks[].number` | ✅ | Номер задания в экзамене (например `"1"`, `"25"`). |
| `exam_tasks[].name` | ✅ | Описание задания. |
| `exam_tasks[].part` | — | Часть экзамена (`A`, `B`, `C`) или `null`. |
| `mapping` | ✅ | Соответствие заданий и тем. |
| `mapping[].exam_task_id` | ✅ | Идентификатор типа задания (должен совпадать с `exam_tasks[].id`). |
| `mapping[].theme_ids` | ✅ | Список идентификаторов тем (должны совпадать с `themes[].id`). |

## Как подготовить JSON из PDF-кодификатора

1. Откройте PDF кодификатора (например, с сайта ФИПИ).
2. Скопируйте список тем (раздел «Перечень элементов содержания»).
3. Скопируйте список заданий (раздел «Перечень требований к уровню подготовки» или таблицу вариантов).
4. Создайте JSON-файл по схеме выше и сохраните его в эту директорию как `<subject>.v1.json`.
5. Запустите импорт (см. ниже).

Пример готового файла: [`example_subject.v1.json`](example_subject.v1.json).

## Импорт данных в базу

```bash
# Импорт всех JSON-файлов из content/
python scripts/import_codifier.py

# Импорт конкретного файла
python scripts/import_codifier.py content/chemistry.v1.json

# Указать путь к базе данных (по умолчанию используется DB_PATH из database.py)
DB_PATH=path/to/bot.db python scripts/import_codifier.py
```

## Практика по номеру задания

Раздел «🔢 Номера заданий» в меню практики позволяет тренировать конкретные задания ЕГЭ по их номеру.

### Как это работает сейчас

Для каждого номера задания (например, №5) бот использует поле `mapping` из кодификатора, чтобы найти связанные темы (`theme_ids`), а затем выбирает случайное задание из таблицы `tasks` по этим темам.

**Текущие ограничения:**
- Задания в таблице `tasks` пока не размечены полем `exam_task_id` (т. е. не привязаны напрямую к конкретному номеру ЕГЭ).
- Поэтому отбор идёт через темы: `exam_task_id → theme_ids → tasks`.

**Опциональное улучшение (будущий PR):** добавить колонку `exam_task_id` в таблицу `tasks`, чтобы каждое задание явно хранило номер экзаменационного задания. Это позволит фильтровать задания точнее.

### Graceful fallback

- Если у предмета нет кодификатора (таблица `exam_task_types` пуста) — показывается сообщение «пока нет структуры» и кнопка возврата в меню практики.
- Если для конкретного номера задания нет заданий в базе — показывается дружественное сообщение с предложением выбрать другой номер.
